🔑 Google OAuth 설정 방법
신규
  1. Google Cloud Console → 새 프로젝트 생성
  2. OAuth 2.0 클라이언트 ID 생성
  3. 승인된 리디렉션 URI 추가:
    http://localhost:3000/api/auth/callback/google
  4. .env.local에 GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET 추가
  5. NEXTAUTH_SECRET 랜덤 문자열 생성 (openssl rand -base64 32)

신규 기능 2 — Supabase DB 스키마 + 마이그레이션
localStorage에 저장되던 모든 데이터를 Supabase PostgreSQL로 이전합니다. RLS로 사용자별 데이터를 완전히 격리합니다.
📋 데이터 마이그레이션 맵
CRITICAL
현재 (localStorage 키) → Supabase 테이블 주요 컬럼 RLS 정책
'scrap_sheets' scrap_sheets id, user_id, title, items[], created_at user_id = auth.uid()
'lyrics_history' lyrics_history id, user_id, title, genre, lyrics, suno_prompt, starred user_id = auth.uid()
'youtube_channels' youtube_channels id, user_id, channel_id, name, subscriber_count, view_count user_id = auth.uid()
'revenue_songs' revenue_songs id, user_id, title, platform, monthly_streams, revenue user_id = auth.uid()
'schedule_config' schedule_configs id, user_id, days[], time, timezone user_id = auth.uid()
💡 점진적 전환 전략: localStorage 함수를 즉시 삭제하지 않고, Supabase 버전을 db.ts에 별도로 추가한 뒤 페이지별로 순차 전환합니다. 로그인 안 한 경우는 localStorage fallback 유지 → 기존 사용자 데이터 손실 방지.
🔒 RLS (Row Level Security) 핵심
CRITICAL
-- 모든 테이블에 적용하는 기본 패턴
ALTER TABLE lyrics_history ENABLE ROW LEVEL SECURITY;

CREATE POLICY "본인 데이터만" ON lyrics_history
  FOR ALL USING (user_id = auth.uid());
